# Contribution Guidelines

Thank you for your interest in contributing to Awesome Ralph!

## Adding a Resource

Please ensure your pull request adheres to the following guidelines:

- **Search previous suggestions** before making a new one to avoid duplicates.
- **Make an individual pull request** for each suggestion.
- Use the following format: `[Resource Name](link) - Description.`
- Descriptions should be concise and end with a period.
- New categories or improvements to the existing categorization are welcome.
- Check your spelling and grammar.
- Make sure your text editor is set to remove trailing whitespace.
- The pull request should have a useful title.

## Quality Standards

Resources should be:

- **Relevant** - Directly related to Ralph/Ralph Wiggum loops or autonomous AI coding patterns.
- **Useful** - Provide value to developers implementing or learning about Ralph.
- **Maintained** - For code repositories, actively maintained projects are preferred.
- **Accessible** - Links should be publicly accessible (no paywalls where possible).

## Updating Your Pull Request

If asked to make changes to your pull request, please edit your original PR rather than opening a new one.

## Reporting Issues

If you find a broken link or outdated resource, please open an issue or submit a PR to fix it.

Thank you for your contributions!
